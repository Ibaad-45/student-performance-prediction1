# 🎓 Student Performance Prediction System

A full-stack machine learning web application that predicts a student's **final exam score**, **letter grade**, and **pass/fail outcome** from their study habits, attendance, and background — built with **scikit-learn**, **Flask**, and **SQLite**.

## 🧠 Overview

This project demonstrates a complete, production-style ML workflow:

1. **Synthetic dataset generation** — 2,500 realistic student records with genuine statistical relationships (study hours, attendance, and prior scores are the strongest predictors — matching published education research).
2. **Data preprocessing** — missing value imputation, IQR-based outlier capping, label encoding, feature scaling.
3. **Feature engineering** — four derived features (study efficiency, attendance-study index, academic momentum, lifestyle balance).
4. **Exploratory Data Analysis** — correlation heatmaps, score distributions, and relationship plots.
5. **Model training** — three regression models compared (Linear Regression, Random Forest, Gradient Boosting) plus a Random Forest classifier for pass/fail.
6. **Evaluation** — R², MAE, RMSE, accuracy, precision, recall, F1, and a confusion matrix.
7. **A Flask REST API + responsive dashboard** to serve live predictions and visualize model performance.
8. **SQLite persistence** of every prediction made through the web app.

## 🚀 Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/<your-username>/student-performance-prediction.git
cd student-performance-prediction

# 2. Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Generate the dataset
python src/generate_dataset.py

# 5. Train the models (creates models/*.pkl and static/plots/*.png)
python src/train_model.py

# 6. Run the web app
python app.py
```

Then open **http://127.0.0.1:5000** in your browser.

Full step-by-step instructions (including troubleshooting) are in [`docs/INSTALLATION.md`](docs/INSTALLATION.md).

## 🔌 REST API Reference

| Method | Endpoint              | Description                                  |
|--------|-----------------------|-----------------------------------------------|
| GET    | `/api/health`         | Health check                                  |
| POST   | `/api/predict`        | Predict a student's performance               |
| GET    | `/api/history?limit=` | Recent prediction history                     |
| GET    | `/api/stats`          | Aggregate stats (total, avg score, pass rate) |
| GET    | `/api/metrics`        | Saved model evaluation metrics                |
| POST   | `/api/history/clear`  | Clear prediction history                      |

**Example request:**
```bash
curl -X POST http://127.0.0.1:5000/api/predict \
  -H "Content-Type: application/json" \
  -d '{
        "gender": "Female", "age": 19, "study_hours_per_week": 18,
        "attendance_percentage": 91, "previous_exam_score": 78,
        "parental_education": "Masters", "family_income_level": "Medium",
        "internet_access": "Yes", "extracurricular_activities": "Yes",
        "sleep_hours": 7.5, "tutoring": "Yes", "part_time_job": "No",
        "screen_time_hours": 2.5, "parental_involvement": "High"
      }'
```

**Example response:**
```json
{
  "success": true,
  "data": {
    "predicted_score": 69.35,
    "predicted_grade": "C",
    "pass_fail": "Pass",
    "pass_probability": 98.99,
    "remark": "Satisfactory performance. Consistent effort will help.",
    "pass_threshold": 40
  }
}
```

> **Note on the dataset:** Real per-student datasets (e.g. from UCI/Kaggle) frequently change hosting and licensing terms. This project ships with a **synthetic dataset generator** (`src/generate_dataset.py`) that encodes realistic, literature-backed relationships between study habits and outcomes, so the entire pipeline is 100% reproducible offline. You can swap in any real CSV with the same column names without changing any other code.
